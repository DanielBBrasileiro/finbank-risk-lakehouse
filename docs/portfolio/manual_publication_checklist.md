# Manual Publication Checklist

These actions require the repository owner's GitHub or LinkedIn session and do not block technical validation.

## GitHub

- [ ] Pin `finbank-risk-lakehouse` on the GitHub profile.
- [ ] Confirm that `v1.0.1-portfolio` is shown as the latest release.
- [ ] Confirm that the README architecture SVG and dashboard screenshot render correctly while signed out.

## LinkedIn

- [ ] Publish the recommended English post from `docs/portfolio/linkedin_post.md`.
- [ ] Add the repository or release to the LinkedIn Featured section.
- [ ] Attach the five-page carousel described in `docs/portfolio/linkedin_carousel.md`.
- [ ] Optionally attach a short dashboard recording after confirming that it shows synthetic data only.
- [ ] Verify line breaks, hashtags, link preview and image order before publishing.

## Suggested Attachment Order

1. Carousel cover.
2. Architecture overview.
3. Credit-risk dashboard.
4. dbt lineage.
5. GitHub Actions summary.
