# Roadmap

## v1.0.0 Portfolio Release

- [x] Generate deterministic synthetic banking data and offline BCB/CVM fixtures.
- [x] Enforce typed, domain and range contracts with a Rust CLI.
- [x] Materialize local Bronze, Silver and Gold Parquet layers.
- [x] Load DuckDB and PostgreSQL without replacing warehouse constraints.
- [x] Build documented dbt staging, intermediate and mart layers.
- [x] Test grain, relationships, domains, reconciliations and business rules.
- [x] Process suspicious events idempotently and verify replay behavior.
- [x] Serve four responsive Streamlit views with graceful optional-data states.
- [x] Govern analytical AI with retrieval boundaries, read-only SQL and audit logs.
- [x] Reuse one executable pipeline from Make, Dagster and containerized Airflow.
- [x] Validate Terraform, Compose, Airflow image and dependency security in CI.
- [x] Publish architecture, test plan, screenshots, demo script and honest scope.
- [x] Verify the full DuckDB demo and PostgreSQL integration from clean runners.

## v1.0.1 Publication Closeout

- [x] Align package metadata and public technical scope.
- [x] Consolidate the architecture diagram into one reusable Mermaid source.
- [x] Add release metrics to the generated evidence pack.
- [x] Standardize LinkedIn, CV, carousel and release materials.
- [x] Publish the validated `v1.0.1-portfolio` tag and release.

## Post-v1 Evolution

- [ ] Add scheduled ingestion with source freshness alerts and operational SLIs.
- [ ] Publish dbt docs as a versioned GitHub Pages artifact.
- [ ] Add OpenLineage-compatible run metadata across orchestrators.
- [ ] Benchmark larger synthetic volumes and document cost/performance tradeoffs.
- [ ] Deploy one cloud target only when credentials, budget and monitoring are available.

Cloud deployment, production SLAs, regulated-model validation and real customer data are intentionally outside the v1 portfolio claim.
